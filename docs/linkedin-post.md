# LinkedIn Post Draft

I recently completed an end-to-end machine-learning project: a **Student Math Score Predictor**.

Instead of stopping at a notebook, I rebuilt the full workflow—from validating raw data and preparing features to comparing regression models, saving the best pipeline, testing the code, and connecting it to a Flask web application.

What the project does:

It estimates a math score from reading and writing results plus learning-context information. I deliberately excluded gender and race/ethnicity from prediction, even though they exist in the source dataset, because sensitive demographic attributes should not determine an individual educational prediction.

My main learning takeaways:

- A preprocessing pipeline keeps training and real-world predictions consistent.
- Models should be compared on the same test data with clear metrics such as MAE, RMSE, and R².
- A useful ML portfolio project needs validation, tests, documentation, and reproducible setup—not only model accuracy.
- Responsible feature selection matters as much as technical feature availability.

The project includes a responsive Flask interface, automated tests, GitHub Actions, Docker support, and complete setup documentation.
